完整流程：GitHub Actions 实现「私有仓源文件加密 → 公共仓部署 GitHub Pages」

该流程核心是：私有仓存源文件→GitHub Actions自动加密→推加密文件到公共仓→公共仓部署Pages，同时确保AES加密安全，以下是全步骤+代码。

一、前期准备（2个仓库+1组密钥）

  1. 创建两个仓库

• 私有仓库（源文件仓）：命名如 vuepress-source-private，存放 VuePress 原始源文件（含未加密的 MD 文档、组件等），仅自己可访问。

• 公共仓库（Pages仓）：命名如 vuepress-pages-public，用于存放加密后的文件，开启 GitHub Pages（设置路径为 main 分支 / 目录）。

  1. 生成 SSH 密钥对（用于仓库间推送）

• 本地终端执行命令生成密钥（替换邮箱为你的GitHub邮箱）：

ssh-keygen -t ed25519 -C "1026619596@qq.com" -f github-actions-deploy-key

• 生成两个文件：github-actions-deploy-key（私钥）、github-actions-deploy-key.pub（公钥）。

  1. 配置密钥权限

• 公共仓（Pages仓）配置公钥：

  1. 进入公共仓 → Settings → Deploy keys → Add deploy key。

  2. Title 填「GitHub Actions Deploy」，Key 粘贴 github-actions-deploy-key.pub 内容，勾选「Allow write access」（允许写入），点击 Add key。

• 私有仓（源文件仓）配置私钥：

  1. 进入私有仓 → Settings → Secrets and variables → Actions → New repository secret。

  2. Name 填 PRIVATE_DEPLOY_KEY，Secret 粘贴 github-actions-deploy-key 内容，点击 Add secret。

  3. 配置加密密钥（AES 密钥/IV）

• 私有仓添加两个 Secret（用于 AES 加密，避免硬编码）：

  1. 新增 Secret：AES_KEY（AES 密钥，建议 16/24/32 位，对应 128/192/256 位加密，推荐 32 位）。

  2. 新增 Secret：AES_IV（AES 偏移量，固定 16 位，如 a1b2c3d4e5f6g7h8）。

  1. 私有仓目录结构（示例）

  1. 关键文件代码 （1）PasswordProtected.vue
<template>
  <div class="password-protected">
    <!-- 密码输入弹窗 -->
    <div class="modal" v-if="showModal">
      <div class="modal-content">
        <h3>请输入访问密码</h3>
        <input
          type="password"
          v-model="inputPassword"
          placeholder="输入密码后查看内容"
          class="password-input"
        />
        <div class="btn-group">
          <button @click="verifyPassword" class="confirm-btn">确认</button>
          <button @click="inputPassword = ''" class="reset-btn">清空</button>
        </div>
        <p class="error-tip" v-if="errorTip">{{ errorTip }}</p>
        <p class="count-tip">剩余尝试次数：{{ remainingTimes }}</p>
      </div>
    </div>

    <!-- 解密成功后渲染内容 -->
    <div class="content" v-if="decryptedContent">
      <!-- 渲染解密后的Markdown内容 -->
      <div v-html="renderedContent"></div>
    </div>
  </div>
</template>

<script>
// 引入AES解密所需依赖（需先安装crypto-js）
import CryptoJS from 'crypto-js';
// 引入VuePress的Markdown渲染工具（用于渲染解密后的Markdown内容）
import { marked } from 'marked';

export default {
  name: 'PasswordProtected',
  props: {
    // 加密后的内容（从MD文件传入）
    encryptedContent: {
      type: String,
      required: true
    },
    // AES加密时的偏移量IV（与加密时一致，从MD文件传入）
    iv: {
      type: String,
      required: true,
      default: 'a1b2c3d4e5f6g7h8' // 默认值，可根据实际加密IV修改
    }
  },
  data() {
    return {
      showModal: true, // 控制密码弹窗显示
      inputPassword: '', // 用户输入的密码
      decryptedContent: '', // 解密后的原始Markdown内容
      renderedContent: '', // 渲染后的HTML内容
      errorTip: '', // 错误提示
      remainingTimes: 3 // 密码尝试次数限制
    };
  },
  methods: {
    // 验证密码并解密内容
    verifyPassword() {
      if (!this.inputPassword) {
        this.errorTip = '请输入密码！';
        return;
      }

      try {
        // 1. 配置AES解密参数（与加密时一致，如模式CBC、填充方式Pkcs7）
        const key = CryptoJS.enc.Utf8.parse(this.inputPassword); // 密码作为密钥
        const iv = CryptoJS.enc.Utf8.parse(this.props.iv); // 传入的IV

        // 2. 执行解密
        const decrypted = CryptoJS.AES.decrypt(
          this.encryptedContent,
          key,
          { iv, mode: CryptoJS.mode.CBC, padding: CryptoJS.pad.Pkcs7 }
        );

        // 3. 转换解密结果为字符串（原始Markdown内容）
        this.decryptedContent = decrypted.toString(CryptoJS.enc.Utf8);

        // 4. 解密成功：隐藏弹窗，渲染Markdown内容
        if (this.decryptedContent) {
          this.showModal = false;
          this.renderedContent = marked(this.decryptedContent); // 将Markdown转为HTML
          this.errorTip = '';
        } else {
          this.handleError();
        }
      } catch (err) {
        // 解密失败（密码错误等）
        this.handleError();
      }
    },

    // 处理密码错误逻辑
    handleError() {
      this.remainingTimes--;
      this.errorTip = '密码错误，请重新输入！';
      this.inputPassword = '';

      // 尝试次数用尽，刷新页面重置
      if (this.remainingTimes <= 0) {
        this.errorTip = '尝试次数已用尽，页面将刷新！';
        setTimeout(() => window.location.reload(), 2000);
      }
    }
  },
  mounted() {
    // 初始化Markdown渲染配置（可选，根据需要调整）
    marked.setOptions({
      breaks: true,
      gfm: true,
      tables: true
    });
  }
};
</script>

（3）encrypt-script.js（AES 加密脚本，自动加密需保护的 MD 文件）

const fs = require('fs');
const path = require('path');
const CryptoJS = require('crypto-js');

// 从环境变量获取 AES 密钥和 IV（GitHub Actions 中注入）
const AES_KEY = process.env.AES_KEY;
const AES_IV = process.env.AES_IV;

// 需加密的文件路径（可根据实际修改）
const TARGET_FILE = path.join(__dirname, 'src', '秘密文档.md');
// 加密后的文件输出路径（覆盖原文件，或输出到临时目录）
const OUTPUT_FILE = TARGET_FILE;

// 读取原始 MD 内容（需加密的部分）
const rawContent = `# 产品核心规划方案
- 2024 Q4 核心目标：用户增长 50%
- 核心功能：XXX 模块迭代（细节略）
- 预算：XXX 万`; // 实际场景可改为读取文件，如 fs.readFileSync('原始秘密内容.md', 'utf8')

// AES 加密函数（CBC 模式，与之前组件解密逻辑一致）
function aesEncrypt(content, key, iv) {
  const keyUtf8 = CryptoJS.enc.Utf8.parse(key);
  const ivUtf8 = CryptoJS.enc.Utf8.parse(iv);
  const encrypted = CryptoJS.AES.encrypt(
    content,
    keyUtf8,
    { iv: ivUtf8, mode: CryptoJS.mode.CBC, padding: CryptoJS.pad.Pkcs7 }
  );
  return encrypted.toString();
}

// 生成加密后的 MD 内容（组件引用+加密串）
const encryptedContent = aesEncrypt(rawContent, AES_KEY, AES_IV);
const finalMdContent = `# 【加密文档】产品核心规划方案

<PasswordProtected 
  :encryptedContent="\`${encryptedContent}\`"
  :iv="\`${AES_IV}\`"
/>

::: warning 访问提示
本页面需输入密码查看，密码请联系管理员获取。
:::
`;

// 写入加密后的 MD 文件
fs.writeFileSync(OUTPUT_FILE, finalMdContent, 'utf8');
console.log(`✅ 加密完成！文件已输出到：${OUTPUT_FILE}`);

（4）encrypt-deploy.yml（GitHub Actions 工作流，自动加密+推送）

name: 加密源文件并部署到 Pages 仓

# 触发条件：私有仓 main 分支推送时执行
on:
  push:
    branches: [ main ]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      # 1. 拉取私有仓源文件
      - name: 拉取私有仓代码
        uses: actions/checkout@v4

      # 2. 配置 Node 环境（匹配 VuePress 需求）
      - name: 配置 Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 18
          cache: 'npm'

      # 3. 安装依赖
      - name: 安装依赖
        run: npm install

      # 4. 执行 AES 加密脚本（加密需保护的 MD 文件）
      - name: 加密源文件
        env:
          AES_KEY: ${{ secrets.AES_KEY }}  # 从私有仓 Secret 读取
          AES_IV: ${{ secrets.AES_IV }}    # 从私有仓 Secret 读取
        run: node encrypt-script.js

      # 5. 构建 VuePress（生成静态 HTML 文件）
      - name: 构建 VuePress
        run: npx vuepress build src

      # 6. 配置 SSH（用于推送到公共仓）
      - name: 配置 SSH 密钥
        run: |
          mkdir -p ~/.ssh/
          echo "${{ secrets.PRIVATE_DEPLOY_KEY }}" > ~/.ssh/id_ed25519
          chmod 600 ~/.ssh/id_ed25519
          ssh-keyscan -H github.com >> ~/.ssh/known_hosts

      # 7. 拉取公共仓代码，替换为加密后的构建产物
      - name: 拉取公共仓并推送构建产物
        run: |
          # 克隆公共仓（替换为你的公共仓地址）
          git clone git@github.com:your-github-username/vuepress-pages-public.git public-repo
          # 删除公共仓原有内容（保留 .git 目录）
          cd public-repo
          rm -rf *
          # 复制 VuePress 构建产物（dist 目录）到公共仓
          cp -r ../src/.vuepress/dist/* .
          # 提交并推送
          git config --global user.name "GitHub Actions"
          git config --global user.email "your-github-email@xxx.com"
          git add .
          git commit -m "Auto deploy: $(date +%Y-%m-%d %H:%M:%S)"
          git push origin main

三、执行流程与验证

  1. 触发工作流

• 向私有仓 main 分支推送代码（如修改 encrypt-script.js 中的原始内容），GitHub Actions 会自动触发 encrypt-deploy.yml 工作流。

  1. 验证结果

• 公共仓：工作流执行完成后，公共仓会新增 VuePress 构建后的静态文件（含加密后的页面）。

• GitHub Pages：访问公共仓的 GitHub Pages 地址（如 https://your-github-username.github.io/vuepress-pages-public/），点击「秘密文档」，会弹出密码框，输入正确的 AES_KEY（即私有仓的 AES_KEY Secret 值），即可解密查看内容。

四、AES 加密安全性说明

  1. 安全性保障：

◦ AES-256 是国际公认的安全加密算法，目前无高效破解方法，暴力破解需尝试 2²⁵⁶ 种可能，几乎不可能。

◦ 本方案中，AES_KEY 和 AES_IV 存储在私有仓 Secret 中，不暴露在代码或前端；加密过程在 GitHub Actions 服务器执行，不经过客户端，进一步降低泄露风险。

  1. 注意事项：

◦ 务必保管好 AES_KEY（丢失则无法解密内容），建议定期更换。

◦ 避免使用 ECB 模式（本方案用 CBC 模式，需搭配 IV 增强安全性）。
